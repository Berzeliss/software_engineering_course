## Part B

### Clean Code Developmenet Cheat Sheet

#### 1. Use Meaningful Names
Variable and function names should clearly describe their function or purpose.

#### 2. Keep Functions Small
Functions should be small and focused on a single task.

#### 3. Use Single Responsibility Principle
Functions should have a single responsibility.

#### 4. Avoid Code Duplication
Reusable code should be extracted into functions.

#### 5. Use Early Returns
Functions should use early returns to handle invalid input and avoid deep nesting.

#### 6. Keep Code Readable
Code should be easy to read and understand, without exessive comments.

#### 7. Use Clear Error Handling
Functions should handle errors gracefully and log useful information.  

#### 8. Use Constants instead of Magic Numbers
Constants should be used instead of magic numbers in code.

#### 9. Well-Structured Repository
The repository should have a clear folder structure and separate files for different types of code.

#### 10. Use Documentation
Functions and classes should have clear documentation.

#### 11. Keep Format Consistent
Code should be formatted consistently across the repository.

#### 12. Use version control
The repository should be managed using version control, such as Git.
